# D3 GUI 需求文档 — 图形化用户界面

> 版本：v1.1  
> 创建日期：2026-05-01  
> 状态：已确认

---

## 1. 定位与目标

D3 在 D1 核心引擎之上，提供一套**图形化桌面应用界面**，让用户通过鼠标操作完成任务的增删改查、日历浏览、搜索等操作。

| 设计原则 | 说明 |
|----------|------|
| 日历为主体 | 月度日历占据页面核心区域，所有操作围绕日历展开 |
| 零菜单栏 | 不设传统菜单栏，所有功能入口内嵌在界面元素中 |
| 中文优先 | 所有文案、提示、按钮文字均为中文 |
| 响应式适配 | 适配 1080P / 2K / 4K 三种主流分辨率 |
| 共用引擎 | 直接 `import` D1 引擎，与 D2 CLI 共享同一数据文件 |
| 暗色模式 | 支持明/暗双主题，跟随系统或手动切换 |

---

## 2. 分辨率适配

| 分辨率 | 典型尺寸 | 日历单元格最小高度 | 日历单元格宽度 | 一次可见任务数（每格） |
|--------|----------|--------------------|----------------|------------------------|
| 1080P | 1920×1080 | ~70px | 等分7列 | 6条（需滚轮查看更多） |
| 2K | 2560×1440 | ~100px | 等分7列 | 6条（需滚轮查看更多） |
| 4K | 3840×2160 | ~140px | 等分7列 | 6条（需滚轮查看更多） |

> **设计策略**：日历区域占窗口高度的 60%，下方详情区占 40%。单元格内始终最多同时展示 6 条任务标题，超出部分通过鼠标滚轮纵向滚动查看。同一天内的任务按**创建时间戳升序**排列。

### 3.1 主题切换按钮

- 位置：搜索栏左侧，页面最顶部
- 图标：月亮（🌙）/太阳（☀️），随当前主题切换
- 点击行为：在明/暗主题间切换
- 默认行为：跟随系统主题设置
- 用户手动切换后：记住用户选择，不再跟随系统（直到下次手动切换为"跟随系统"）

### 3.2 搜索栏

- 位置：页面最顶部，主题切换按钮右侧，横跨剩余宽度
- 左侧为文本输入框，placeholder 文字 "搜索任务..."
- 右侧尾部有一个放大镜图标按钮
- 支持键盘 Enter 触发搜索（详见快捷键 4.6）
- 搜索结果以下拉弹窗展示（详见 4.4）

### 3.3 年月导航

- 位置：搜索栏下方，日历上方
- 中央显示 "2026年 5月"（年份+月份）
- **点击"5月"区域**：弹出下拉弹窗，固定展示 12 个月份（1月~12月），点击某个月份即切换
- **点击"2026年"区域**：弹出下拉弹窗，每页展示 5 个年份，可通过翻页按钮切换页码范围，点击某个年份即切换
- 年份范围：当前年份 ±20 年（如 2026 年则范围 2006~2046）
- 左侧箭头按钮（◀）：点击切换到上一月
- 右侧箭头按钮（▶）：点击切换到下一月
- 边界行为：超出 ±20 年范围时箭头置灰不可点击

### 3.4 日历主体

- 位置：页面中央，占窗口高度约 60%
- 7 列表格：一 / 二 / 三 / 四 / 五 / 六 / 日
- 每个日期格：
  - 左上角显示日期数字
  - 今天的日期数字后附加**蓝色小圆标（●）**和**小字体"今天"**文字
  - 下方是任务列表区，最多同时展示 6 条任务的 title
  - 超出 6 条时，单元格内出现纵向滚动条，鼠标滚轮可翻阅
  - 即使当天无任务，单元格也保持统一高度
  - 任务按创建时间戳升序排列
- 任务 title bar：
  - 带颜色状态指示（详见 3.7）
  - 鼠标悬停时，bar 尾部出现**工具按钮（编辑）**、**红色叉按钮（删除）**和**加号按钮（新建子任务，仅父任务显示）**
- 非当月日期（前月末尾/后月开头）：以灰色显示，但仍可放置任务

#### 3.4.1 子任务展示

- 子任务与主任务混排在同一日期的任务列表中
- 子任务以以下视觉特征与主任务区分：
  - **缩进**：向右缩进，前面带 `└─` 图标
  - **虚线边框**：task bar 使用虚线左边框
  - **更浅的颜色**：底色比同状态主任务浅一个色阶
  - 文字使用**斜体或更纤细的字体**
- 排序规则：同一日内所有任务（主任务+子任务）按创建时间戳升序排列
- 子任务鼠标悬停时仅显示工具按钮和删除按钮，不显示加号按钮

### 3.5 详情文本区

- 位置：日历下方，等宽区域，占窗口高度约 40%
- **详情区永不空白**：始终展示内容
- **初始状态**（软件打开时）：默认展示**今天**的所有任务（含子任务）详情
  - 今天的日期单元格高亮突出显示
  - 各任务按规则排列（子任务紧随其父任务后缩进展示，见 3.5.1）
- 当**选中日历中某一条任务**时（单击 task title bar）：
  - 该 task bar 在日历中**高亮突出显示**（边框加粗/背景加深）
  - 详情区切换到展示该任务的 5 个字段：**标题、开始日期、截止日期、状态、背景**
  - 标题字段 bar 尾部有**工具按钮（✏️ 编辑）**、**红色叉按钮（❌ 删除）**和**加号按钮（➕ 新建子任务，仅父任务显示）**
  - 详情区右上角显示红色 **"新建"** 按钮
  - 子任务详情中额外显示**所属任务标题**
- 当**单击日历中某一天的空白区域**时：
  - 该日期单元格**高亮突出显示**
  - 详情区切换到展示该天所有任务（含子任务）的详情列表
  - 每条任务的标题字段 bar 尾部有工具按钮和删除按钮（父任务额外有加号按钮）
  - 详情区右上角显示红色 **"新建"** 按钮
- 当**单击日历中另一天**时（无论点击空白还是任务）：
  - 详情区**直接切换到**该天的内容（选中任务则显示单任务详情，点击空白则显示该天全部任务）
- 当**点击日历或详情区以外的空白区域**时：详情区内容**维持不变**

#### 3.5.1 详情区子任务排列

- 展示某天所有任务时，子任务**紧随其父任务之后缩进展示**
- 使用**斜体或更纤细的字体**与父任务区分
- 每个子任务显示 5 个字段，标题字段 bar 尾部有工具按钮和删除按钮

### 3.6 "新建"按钮

- 位置：详情区右上角（始终可见，因为详情区永不空白）
- 样式：红色按钮，文字"新建"
- 点击行为：弹出新建任务弹窗（详见 4.1）
- 如果当前详情区展示的是某一天的任务列表（由点击日历空白触发），新任务的 `start_date` 自动填入该日期；否则默认为今天

### 3.7 任务颜色状态

| 状态 | 主任务颜色 | 子任务颜色 | 含义 |
|------|-----------|-----------|------|
| 未启动 | 灰色 (#9CA3AF) | 浅灰 (#D1D5DB) | 尚未开始 |
| 完成中 | 蓝色 (#3B82F6) | 浅蓝 (#93C5FD) | 正在进行 |
| 已完成 | 绿色 (#10B981) | 浅绿 (#6EE7B7) | 已经完成 |
| 已取消 | 灰色 + 删除线 | 浅灰 + 删除线 | 不再执行 |

> 任务 title bar 使用左边框色 + 浅底色来展示状态颜色。子任务使用虚线边框 + 更浅底色，并在日历中缩进展示。

## 4. 动态交互

### 4.1 新建任务

**触发方式**：
- 单击日历中某天的空白区域，再点击详情区右上角"新建"按钮
- 直接点击详情区右上角"新建"按钮
- 键盘快捷键 `Ctrl+N`（详见 4.6）

**弹窗行为**：
- 在页面正中央弹出模态对话框（带半透明遮罩层）
- 弹窗标题："新建任务"
- 弹窗底色：白色（明色模式）/ 深色（暗色模式）
- 展示 5 个可编辑字段：

| 字段 | 输入方式 | 默认值 |
|------|----------|--------|
| 标题 | 文本输入框 | 空 |
| 开始日期 | 日期选择器（日历弹窗） | 触发时所在日期格（若从日历触发），否则为今天 |
| 截止日期 | 日期选择器（日历弹窗） | 与开始日期相同 |
| 状态 | 单选按钮组：未启动 / 完成中 / 已完成 / 已取消 | 未启动 |
| 背景 | 多行文本输入框（≤500字） | 空 |

- 底部按钮：
  - **保存**：校验标题非空 → 调用 `task_manager.create_task()` → 关闭弹窗 → 刷新日历
  - **取消**：关闭弹窗，不保存
- 校验失败时在弹窗内显示红色错误提示

#### 4.1.1 新建子任务

**触发方式**：
- 鼠标悬停在日历中的父任务 title bar → bar 尾部出现加号按钮（➕）→ 点击加号按钮
- 在详情区选中某父任务 → 标题字段 bar 尾部出现加号按钮 → 点击加号按钮

**弹窗行为**：
- 弹窗标题："新建子任务"
- 弹窗底色：相比父任务弹窗**稍偏灰色**，用于与父任务弹窗区分
- 底部显示："所属任务：<父任务标题>"
- 字段内容和交互与新建任务弹窗一致
- 保存时调用 `task_manager.add_subtask()`

### 4.2 编辑任务

**触发方式**：
- 鼠标悬停在日历中的 task title bar → bar 尾部出现工具按钮 → 点击工具按钮
- 在详情区中点击标题 bar 尾部的工具按钮

**弹窗行为**：
- 弹窗标题："编辑任务"
- 各字段预填当前值
- 字段修改后点击**保存** → 调用 `task_manager.update_task()` → 刷新日历和详情区
- 点击**取消** → 关闭弹窗，不保存

**编辑子任务**：同上，弹窗标题改为"编辑子任务"，底部显示"所属任务：XXX"

### 4.3 删除任务

**触发方式**：
- 鼠标悬停在日历中的 task title bar → bar 尾部出现红色叉按钮 → 点击红色叉按钮
- 在详情区中点击标题 bar 尾部的红色叉按钮
- 键盘快捷键 `Delete`（需先选中任务，详见 4.6）

**弹窗行为**：
- 弹窗标题："确认删除"
- 内容：`确认删除任务 "<任务标题>"？`
- 如果是父任务且包含子任务，追加提示：`（包含 N 条子任务，将一并删除）`
- 按钮：
  - **确认删除**（红色）：调用 `task_manager.delete_task()` → 关闭弹窗 → 刷新日历，详情区切换到当天视图
  - **取消**：关闭弹窗，不删除

**删除子任务**：同上，弹窗内容替换为子任务标题，不包含子任务数量提示

### 4.4 搜索任务

**触发方式**：
- 在搜索框输入字符后，点击放大镜按钮
- 在搜索框输入字符后，按键盘 Enter
- 键盘快捷键 `Ctrl+F` 聚焦搜索框（详见 4.6）

**弹窗行为**：
- 在下拉弹窗中展示搜索结果（紧贴搜索框下方展开）
- 每条结果展示：
  - **任务标题**（主显示）
  - **开始日期**（`start_date`，以较小字体或灰色显示在标题右侧）
  - **任务背景**（截断至 50 字，第二行展示）
- 每页展示 10 条记录
- 超过 10 条时，弹窗内可通过鼠标滚轮下翻查看更多
- 匹配规则（复用 D2 CLI `search` 逻辑）：
  - 标题 + 背景中模糊匹配
  - ≥2 个字符部分匹配
  - 大小写不敏感
  - 不显示已删除任务
- 结果列表底部显示总数：`共 N 条结果`

**选中结果**：
- 用户点击某条结果 → 日历自动切换到该任务的 `start_date` 所属月份
- 详情区自动展示该任务详情
- 搜索下拉弹窗关闭

### 4.5 撤销删除（Undo）

- 删除任务/子任务后，在详情区或日历中该位置短暂显示一条提示：
  - `"Q2原型开发" 已删除 [撤销]`
  - 点击"撤销"链接 → 调用 `task_manager.undo_task()` → 恢复任务 → 刷新日历
- 提示显示 5 秒后自动消失，消失后无法通过界面撤销（需通过 CLI `todo undo`）

### 4.6 键盘快捷键

快捷键**仅在 Todo Manager 窗口处于活动状态（聚焦）时生效**，不是全局快捷键。

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+N` | 新建任务 | 弹出新建任务弹窗，`start_date` 默认值为当前详情区关联的日期 |
| `Ctrl+F` | 聚焦搜索框 | 光标跳转到搜索框，可直接输入搜索关键词 |
| `Delete` | 删除任务/子任务 | 需当前已选中某任务（详情区正在展示该任务），弹出确认删除弹窗 |
| `Ctrl+E` | 编辑任务/子任务 | 需当前已选中某任务，弹出编辑弹窗 |
| `←` | 上一月 | 日历切换到上一月 |
| `→` | 下一月 | 日历切换到下一月 |
| `Esc` | 关闭弹窗 | 关闭当前任何打开的弹窗或下拉面板 |

### 4.7 暗色模式

- 应用支持**明色模式**和**暗色模式**
- 默认行为：跟随操作系统的主题设置
- 手动切换：点击搜索栏左侧的主题切换按钮（🌙/☀️）
- 切换行为：
  - 点击按钮立即切换主题
  - 切换后记住用户选择，后续启动使用用户上次选择的主题
  - 如需恢复"跟随系统"，在主题切换按钮上右键点击选择"跟随系统"
- 暗色模式下的颜色调整：
  - 日历背景：深色
  - 任务 bar 底色：降低亮度，保持颜色辨识度
  - 详情区背景：深色
  - 弹窗背景：深色

## 5. 数据流与状态

```
User Action
    │
    ▼
┌──────────┐   调  用   ┌──────────────┐
│  GUI 层  │ ────────→  │  D1 Engine   │
│ (PySide6)│ ←──────── │ task_manager │
└──────────┘   返  回   └──────┬───────┘
                              │
                              ▼
                      ┌──────────────┐
                      │  storage.py  │
                      │ (JSON 文件)  │
                      └──────────────┘
```

- GUI 层通过 `task_manager.TaskManager` 实例与 D1 引擎交互
- 所有数据变更（新建/编辑/删除）立即写入 JSON 文件
- 日历展示数据通过 `calendar_utils.get_month_grid()` 获取
- GUI 不维护独立数据缓存，所有展示数据从 engine 实时读取
