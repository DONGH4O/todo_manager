# 设计系统 v2

> 对应里程碑：M4 GUI/UX 重设计  
> 日期：2026-05-20  
> 参考原型：`docs/m4_uiux_redesign/prototype_v1.html`

## 1. 设计方向

设计系统 v2 采用现代、扁平、轻科技感的桌面应用风格。主背景使用浅蓝、浅绿到浅银的渐变，界面控件以半透明浅色面板、低饱和描边和清晰状态色区分层级。

关键词：

- 清爽。
- 稳定。
- 可长时间使用。
- 月历优先。
- 状态明确。

## 2. 颜色

### 2.1 明色主题

| Token | 色值 | 用途 |
|---|---|---|
| `bg-a` | `#DFF4FF` | 背景渐变起点，浅蓝 |
| `bg-b` | `#DFF8EC` | 背景渐变中段，浅绿 |
| `bg-c` | `#EEF2F5` | 背景渐变终点，浅银 |
| `ink` | `#123047` | 主文本 |
| `muted` | `#6A7D8E` | 辅助文本 |
| `soft` | `rgba(255,255,255,0.68)` | 顶部栏背景 |
| `surface` | `rgba(255,255,255,0.84)` | 主面板背景 |
| `line` | `rgba(98,132,154,0.18)` | 默认描边 |
| `line-strong` | `rgba(73,121,151,0.34)` | hover / selected 描边 |
| `blue` | `#2E8DF5` | 主操作、选中状态 |
| `blue-soft` | `#E8F4FF` | 完成中状态背景 |
| `green` | `#23B883` | 已完成状态 |
| `green-soft` | `#E8F8F0` | 已完成状态背景 |
| `silver` | `#EEF3F6` | 未启动和中性背景 |
| `danger` | `#E05858` | 删除等危险操作 |
| `danger-soft` | `#FFF0F0` | 危险操作背景 |

### 2.2 状态色

| 任务状态 | 主色 | 背景 | 文本表现 |
|---|---|---|---|
| 未启动 | `#98A8B6` | `silver` | 正常文本 |
| 完成中 | `blue` | `blue-soft` | 正常文本 |
| 已完成 | `green` | `green-soft` | 正常文本 |
| 已取消 | `#AEB8C2` | `#EDF1F4` | 任务摘要可加删除线 |

### 2.3 暗色主题方向

M4 原型聚焦明色主题，M5 实现必须补齐暗色主题，并保持同一 token 语义。

建议暗色映射：

| Token | 建议色值 | 用途 |
|---|---|---|
| `bg-root` | `#07131D` | 根背景 |
| `surface` | `rgba(18,35,48,0.92)` | 面板背景 |
| `surface-soft` | `rgba(26,47,62,0.82)` | 顶部栏和输入背景 |
| `ink` | `#E8F3F8` | 主文本 |
| `muted` | `#94A8B7` | 辅助文本 |
| `line` | `rgba(146,180,198,0.18)` | 默认描边 |
| `blue` | `#65B5FF` | 主操作 |
| `green` | `#4DD7A1` | 已完成 |
| `danger` | `#FF7A7A` | 危险操作 |

暗色主题要求：

- 不改变控件位置和工作流。
- 所有文本对比度必须足以长时间阅读。
- 状态色在暗色背景下仍能区分。
- 主题持久化优先使用 Qt 原生设置机制。
- 主题入口采用 `自动 / 明色 / 暗色` 三段式控件，默认自动跟随系统；切换后应立即反馈，并保存用户选择。

## 3. 字体

字体族：

- Windows：`Microsoft YaHei` 优先。
- macOS：`PingFang SC` 优先。
- Linux / fallback：`Noto Sans CJK SC`、系统 sans-serif。

字号：

| Token | 大小 | 用途 |
|---|---:|---|
| `font-xs` | 12px | 标签、辅助说明、日期摘要 |
| `font-base` | 14px | 表单、按钮、任务正文 |
| `font-md` | 16px | 面板标题 |
| `font-lg` | 17px | 产品标题 |
| `font-xl` | 21px | 统计数字 |

规则：

- 不使用随视口宽度变化的字体。
- 字间距保持 `0`。
- 面板内部标题不使用过大的 hero 级字体。

| `space-1` | 4px | 输入框内按钮偏移、小间距 |
| `space-2` | 6px | 分段控件间距 |
| `space-3` | 8px | 任务标签、日历格间距 |
| `space-4` | 10px | 列表项内边距 |
| `space-5` | 12px | 面板内边距 |
| `space-6` | 14px | 主布局间距 |

尺寸：

|---|---|
| 顶部按钮 | 44px 高 |
| 搜索输入框 | 44px 高 |
| 搜索框内按钮 | 36px 高 |
| 日历面板头部 | 62px 高 |

## 5. 阴影与层级

阴影应克制，避免卡片套卡片的厚重感。

| 层级 | 用法 |
|---|---|
| 根背景 | 无阴影，渐变 + 轻网格纹理 |
| 主面板 | 轻阴影 `0 18px 50px rgba(58,97,124,0.14)` |
| 搜索下拉 | 高层级阴影，必须浮在月历上方 |
| 弹窗 | 最强阴影，背景遮罩 + blur |
| toast | 中等阴影，固定底部居中 |

层级建议：

- 顶部栏高于正文面板。
- 搜索下拉高于月历和详情面板。
- 弹窗高于搜索下拉。
- toast 高于普通面板。

## 6. 组件规范

### 6.1 顶部栏

顶部栏是全局操作区。

要求：

- 搜索框和按钮垂直居中。
- 产品标识、搜索、按钮组保持同一水平秩序。
- 允许窗口窄时折叠为单列，但控件不得重叠。

### 6.2 按钮

按钮类型：

| 类型 | 用途 |
|---|---|
| 普通按钮 | 搜索、回到今天、刷新、取消 |
| 主按钮 | 新建、保存、创建 |
| 危险按钮 | 删除、确认删除 |
| 图标按钮 | 月份上一页、下一页 |
| 分段控件 | 主题模式、状态筛选 |

规则：

- 顶部主要按钮高度 44px。
- 月份箭头按钮可保持 44px 方形。
- 所有按钮必须有 tooltip。
- hover 可轻微变亮或描边加深，不应产生布局位移。
- 主题模式分段控件总高度 44px，内部按钮使用图标加短标签，选中态用清晰底色、描边或焦点色表达。

### 6.3 输入框与选择框

输入控件包括：

- 搜索框。
- 标题输入框。
- 日期输入框。
- 状态选择框。
- 备注文本框。
- 年份和月份选择框。

规则：

- 默认描边使用 `line`。
- focus 使用蓝色描边和轻焦点环。
- 日期和状态输入必须与 engine 校验规则一致。
- 年份选择范围至少覆盖常见历史和未来年份；原型范围为 `1900-2100`。

### 6.4 搜索下拉

搜索下拉是顶部栏的一部分，不属于月历面板。

布局：

- 顶部显示搜索结果摘要。
- 列表项左侧为标题、日期区间、备注摘要。
- 右侧为状态胶囊。
- 高度使用 `min(560px, viewport)`，内容内部滚动。

状态：

- 聚焦：展示最近任务或说明。
- 输入：展示过滤结果。
- 无结果：展示空状态。
- 点击结果：定位任务。

### 6.5 月历格

日历格必须保持稳定尺寸。

展示内容：

- 日期数字。
- 当天任务数量角标。
- 最多 3 条任务摘要。

状态：

- 当前月外日期降低透明度。
- 今天使用浅蓝到浅绿背景。
- 选中日期使用蓝色描边。
- hover 加深描边和面板底色。

### 6.6 任务列表项

今日节奏列表项展示：

- 状态点。
- 标题。
- 状态胶囊。
- 日期区间。
- 备注摘要。
- 子任务统计。

不展示：

- 项目标签。
- 优先级。
- 登记时间。
- 任何 engine 当前不存在的字段。

### 6.7 状态胶囊

状态胶囊用于主任务和子任务。

规则：

- 四态必须全部支持。
- 已取消可使用删除线，但不能与删除操作混淆。
- 状态文字必须完整显示。

### 6.8 子任务列表

子任务行布局：

- 左侧为标题与日期区间，保持单行标题截断。
- 右侧为紧凑状态选择胶囊，宽度约 88-96px。
- 状态选择胶囊使用原生下拉或等价 Qt compact combobox，可选择 `未启动`、`完成中`、`已完成`、`已取消`。

滚动：

- 子任务过多时仅子任务列表内部滚动。
- 任务详情整体也允许内部滚动。

### 6.9 弹窗

弹窗类型：

- 新建任务。
- 新增子任务。
- 删除确认。

规则：

- 弹窗居中。
- 背景遮罩。
- 输入字段按逻辑从上到下排列。
- 内容过高时弹窗内部滚动。
- 删除确认必须明确任务名称。

### 6.10 Toast

toast 用于短反馈。

类型：

- 普通提示：保存成功、刷新完成，不显示撤销。
- 可撤销提示：删除完成，显示撤销。

规则：

- 删除之外不显示撤销按钮。
- toast 自动消失。
- 仅删除可恢复，因为底层是软删除。

## 7. 任务状态规则

状态枚举必须与 engine 保持一致：

```text
未启动 / 完成中 / 已完成 / 已取消
```

主任务状态可直接编辑。

子任务在 GUI 中使用紧凑状态选择胶囊：

| GUI 操作 | 保存后状态 |
|---|---|
| 选择 `未启动` | 未启动 |
| 选择 `完成中` | 完成中 |
| 选择 `已完成` | 已完成 |
| 选择 `已取消` | 已取消 |

紧凑规则：

- 不使用主任务同款四段分段控件，避免子任务列表过宽。
- 状态文字必须完整显示，不得只靠颜色表达。
- 仅调整选择控件但未点击保存时，不写入 engine。

## 8. 响应式规则

### 8.1 1920x1080

- 三栏布局。
- 月历居中为主视图。
- 今日节奏和详情区均可滚动。
- 顶部栏不换行。

### 8.2 2560x1440

- 保持最大内容宽度，居中。
- 日历格可更舒展，但文字大小不随视口放大。
- 面板间距保持克制。

### 8.3 3840x2160

- 内容不铺满整屏，最大宽度约 1440px。
- 不放大字号。
- 不增加额外装饰面板。

### 8.4 小屏

- 先两列，再单列。
- 单列下日历任务摘要可隐藏或减少。
- 顶部操作按钮可等宽排列。

## 9. 可访问性与可用性

- 所有按钮必须有 tooltip。
- 搜索框需要清晰 placeholder。
- 表单 label 必须与输入控件对应。
- 键盘快捷键不得干扰文本输入。
- 危险操作必须二次确认。
- 颜色不能作为唯一信息表达，状态文字必须存在。

## 10. M5 落地边界

M5 需要把本设计系统映射到 PySide6：

- 使用 `QSettings` 或等价 Qt 原生能力持久化主题。
- 使用集中 theme/token 模块，不在 widget 内散落硬编码颜色。
- GUI 层通过 engine API 修改数据。
- 搜索和列表可以做视图层缓存，但不能替代 engine 数据源。
- 测试重点覆盖核心交互，不追求像素级快照。
