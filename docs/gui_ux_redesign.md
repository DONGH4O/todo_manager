# GUI/UX 重设计说明

> 对应里程碑：M4 GUI/UX 重设计  
> 日期：2026-05-20  
> 原型：`docs/m4_uiux_redesign/prototype_v1.html`  
> 状态：已确认，可进入 M5 GUI 重构实现

## 1. 设计目标

M4 的目标是先固化产品级桌面体验，再进入 PySide6 widget 重构。新 GUI 继续以月历为主体，同时提升信息层级、视觉清晰度和长时间任务管理的舒适度。

核心原则：

- 月历是主工作台，今日任务和任务详情是辅助面板。
- 所有展示字段必须来自现有 engine 数据模型：`title`、`start_date`、`end_date`、`status`、`background`、`subtasks`。
- 不展示没有底层支持的数据，例如项目标签、优先级、登记时间、提醒时间。
- GUI 不维护独立业务缓存，所有数据变更必须通过 engine 完成。
- 删除可撤销；普通保存不提供撤销，避免暗示底层存在未实现的编辑回滚能力。

## 2. 信息架构

主窗口采用三栏布局：

| 区域 | 位置 | 职责 |
|---|---|---|
| 顶部栏 | 全宽顶部 | 产品标识、全局搜索、回到今天、刷新、新建任务、主题模式 |
| 今日节奏 | 左侧 | 当前选中日期的任务列表、任务概览、状态筛选 |
| 月历工作台 | 中间 | 月份/年份导航、按日期展示任务、选中日期 |
| 任务详情 | 右侧 | 主任务编辑、状态切换、子任务查看与新增、保存/删除 |

顶部品牌区显示：

- 主标题：`Todo Manager`
- slogan：`让每个待办都有清晰节奏`

## 3. 布局规则

### 3.1 桌面三栏

默认宽屏布局：

- 左栏：约 260px，展示今日节奏。
- 中栏：自适应，最小约 560px，展示月历。
- 右栏：约 340px，展示任务详情。
- 顶部栏高度固定，正文区高度贴合窗口。

页面整体不得随内容增多而产生 body 级滚动。内容溢出时，各区域自行滚动：

- 今日节奏任务列表内部滚动。
- 搜索结果下拉内部滚动。
- 任务详情正文内部滚动。
- 子任务列表内部滚动。
- 弹窗内容超高时弹窗内部滚动。

### 3.2 1080P / 2K / 4K

| 分辨率 | 规则 |
|---|---|
| 1080P | 三栏保持可用，任务详情字段不溢出；日历格内仅展示有限任务摘要 |
| 2K | 三栏可适度放宽间距，月历仍为视觉中心 |
| 4K | 内容最大宽度约 1440px，居中显示，避免控件被拉得过散 |

### 3.3 窄屏降级

当窗口宽度不足以稳定展示三栏时：

- 先降级为左栏 + 月历两列，任务详情移至下一行。
- 更窄时降级为单列堆叠。
- 单列模式允许正文容器内部滚动，但仍避免控件文字互相遮挡。

## 4. 顶部栏

顶部栏包含产品标识、搜索框和全局操作按钮。

控件顺序：

1. 产品标识。
2. 搜索输入框，placeholder 为“搜索标题、备注、子任务、状态”。
3. 搜索按钮。
4. 回到今天按钮。
5. 刷新按钮。
6. 新建按钮。
7. 主题模式分段控件：自动、明色、暗色。

对齐规则：

- 搜索框、搜索按钮、回到今天、刷新、新建、主题模式控件在同一水平中心线。
- 顶部按钮高度统一为 44px。
- 所有按钮必须有 tooltip。
- 主题模式控件使用三段式胶囊，默认 `自动`，用户明确选择明色或暗色后再覆盖系统偏好。

## 5. 今日节奏

今日节奏展示当前选中日期的任务，不仅限真实当天。区域标题随日期变化，例如 `5月20日`。

任务概览展示：

- 当月任务数量。
- 当日安排数量。
- 未启动数量。
- 完成中数量。
- 已完成数量。
- 已取消数量。

除“当月任务数量”外，其余概览统计均按当前选中日期计算。

状态筛选：

- 全部。
- 未启动。
- 完成中。
- 已完成。
- 已取消。

任务列表项展示：

- 任务标题。
- 任务状态胶囊。
- 开始日期到截止日期。
- 备注摘要。
- 子任务数量。
- 已完成子任务数量。

已取消任务：

- 状态点使用低饱和灰色。
- 标题可加删除线，表示不再执行。

## 6. 月历工作台

月历工作台是主视图。

导航规则：

- 年份使用下拉选择，范围应覆盖较早和较晚年份。原型范围为 `1900-2100`。
- 月份使用下拉选择。
- 左右箭头用于切换上个月和下个月。
- 回到今天按钮将年份、月份、选中日期恢复到今天。

日历格规则：

- 周一到周日排列。
- 每月显示 6 行，保证布局高度稳定。
- 当前月外日期降低透明度。
- 今天使用浅蓝到浅绿背景强调。
- 选中日期使用蓝色描边。
- 日期角标显示当天关联任务数量。
- 日格内展示最多 3 条任务摘要，超出不展开，避免挤压。

任务出现在日历中的规则：

- 当日期落在任务 `[start_date, end_date]` 区间内时展示。
- 子任务不独立出现在月历中，只随父任务在详情区展示。

## 7. 任务详情

任务详情顶端展示：

- 任务日期区间：`start_date - end_date`。
- 当前状态。

详情字段：

- 标题。
- 开始日期。
- 截止日期。
- 备注。
- 主任务状态。
- 子任务列表。

主任务状态为四态：

- 未启动。
- 完成中。
- 已完成。
- 已取消。

状态切换使用分段控件。保存后写入 engine。

## 8. 子任务

子任务字段至少展示：

- 子任务标题。
- 开始日期。
- 截止日期。
- 子任务状态。

为了节省空间，子任务列表不使用主任务同款分段控件。每条子任务右侧使用紧凑状态选择胶囊，固定宽度展示当前状态，点击后可在四态中选择：

- `未启动`
- `完成中`
- `已完成`
- `已取消`

状态保存规则：

- 用户调整子任务状态选择胶囊并点击保存：该子任务状态按所选四态写入 engine。
- 仅切换状态选择但未保存时，不应写入 engine。
- 不用复选框映射状态，避免 `已取消` 被误保存为 `已完成`。

新增子任务：

- 在任务详情的子任务标题行右侧放置 `+ 子任务` 按钮。
- 点击后打开新增子任务弹窗。
- 弹窗字段：标题、开始日期、截止日期、当前状态。

## 9. 搜索

搜索入口在顶部栏中间，支持搜索：

- 主任务标题。
- 主任务备注。
- 主任务状态。
- 主任务日期。
- 子任务标题。
- 子任务状态。
- 子任务日期。

搜索交互：

- 聚焦搜索框时展示最近任务和可搜索字段说明。
- 输入关键字时实时筛选。
- 点击搜索按钮时展开结果。
- 搜索结果显示匹配数量。
- 每条结果显示标题、日期区间、备注摘要、状态。
- 点击结果后定位到任务开始日期所在月份，选中任务并刷新详情区。

层级规则：

- 搜索下拉必须高于月历工作台和其他面板。
- 下拉高度应足够展示多条结果，原型使用 `min(560px, viewport)`。
- 下拉内容过多时内部滚动。

空状态：

- 无结果时展示“没有找到匹配任务。”。

## 10. 新建任务

点击顶部 `+ 新建` 打开新建任务弹窗。

字段：

- 标题。
- 开始日期。
- 截止日期。
- 当前状态。
- 备注。

默认值：

- 开始日期默认当前选中日期。
- 截止日期默认当前选中日期。
- 状态默认 `未启动`。
- 标题为空时原型可使用“未命名任务”；M5 实现应优先遵循 engine 校验规则。

保存后：

- 通过 engine 创建任务。
- 选中新任务。
- 月历跳转到任务开始日期所在月份。

## 11. 删除与撤销

删除任务必须先弹出确认框。

确认后：

- 调用 engine 删除任务。
- 显示 toast：“任务已删除”。
- toast 中显示“撤销”按钮。

撤销：

- 仅删除后可撤销，因为删除是软删除状态。
- 普通保存、新建、刷新不显示撤销按钮。

## 12. 键盘行为

推荐快捷键：

| 快捷键 | 行为 |
|---|---|
| `/` | 聚焦搜索框 |
| `N` | 打开新建任务弹窗 |
| `Esc` | 关闭搜索下拉或当前弹窗 |

规则：

- 快捷键不得干扰输入框和文本区域中的文字输入。
- 弹窗打开时，焦点应进入第一个主要输入框。

## 13. 空状态和边界状态

必须定义以下空状态：

- 选中日期无任务：今日节奏显示“这一天还没有任务。”。
- 未选中任务：任务详情显示“请选择日历中的任务，或新建一个任务。”。
- 搜索无结果：搜索下拉显示“没有找到匹配任务。”。
- 子任务为空：任务详情保留 `+ 子任务` 入口，并显示空列表占位。

边界状态：

- 备注过长时，今日节奏只展示摘要，详情区可滚动。
- 任务过多时，今日节奏列表滚动。
- 子任务过多时，子任务列表滚动。
- 弹窗过高时，弹窗内部滚动。

## 14. 平台差异允许范围

Windows 与 macOS 应保持同一信息架构、控件顺序、字段含义和状态规则。

允许差异：

- 系统字体回退不同。
- 原生日期选择、下拉框、滚动条样式不同。
- macOS 可适度使用更轻的阴影和圆角观感，但不得改变控件位置和工作流。

不允许差异：

- 字段缺失。
- 状态枚举不一致。
- 删除与撤销规则不一致。
- 搜索定位行为不一致。

## 15. M5 实现提示

- 先抽象 GUI 与 engine 的交互层，再重构 widget 外观。
- 任务状态必须使用 `engine.models.VALID_STATUSES`。
- 任务日期使用 `start_date` 和 `end_date`，不要重新引入登记时间。
- 子任务只在父任务详情中展示，不要在月历中独立展示。
- toast 需要区分普通提示和可撤销提示。
- 搜索下拉层级需要高于月历和详情区。
- 主题模式三态入口先以 HTML 原型确认，确认后再进入 PySide6 实现。
